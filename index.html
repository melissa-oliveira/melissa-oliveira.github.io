Principais atividades no GitHub

Veja os conceitos essenciais para gerenciar repositórios e colaborar em projetos.

  • Repositório – Diretório onde o Git armazena o histórico do projeto.
  • Commit – Registra mudanças no repositório com uma mensagem descritiva.
  • Branch – Linha de desenvolvimento paralela dentro do repositório.
  • Merge – Une alterações de diferentes branches.
  • Clone – Copia um repositório remoto para um ambiente local.
  • Pull – Atualiza o repositório local com mudanças do repositório remoto.
  • Push – Envia alterações do repositório local para o remoto.

Como clonar um repositório

Siga os passos abaixo para copiar um repositório remoto para seu ambiente local:

  1. Copie a URL do repositório no GitHub.
  2. Abra o terminal e navegue até o diretório onde deseja salvar o repositório.
  3. Execute o comando git clone URL_DO_REPOSITORIO.
  4. Aguarde o download dos arquivos e entre no diretório clonado com cd nome-do-repositorio.

Como fazer um commit

Siga os passos abaixo para registrar suas mudanças no repositório:

  1. Abra o terminal e navegue até o diretório do seu repositório.
  2. Use o comando git status para verificar quais arquivos foram modificados.
  3. Adicione os arquivos que deseja commitar com git add nome_do_arquivo ou git add . para adicionar todos.
  4. Realize o commit com o comando git commit -m "Mensagem descritiva das mudanças".
  5. Se necessário, envie as alterações para o repositório remoto com git push origin nome-da-branch.

Como utilizar branches e fazer merge

Siga os passos abaixo para criar branches e unir mudanças:

  1. Crie uma nova branch com git branch nome-da-branch.
  2. Alterne para a nova branch com git checkout nome-da-branch ou git switch nome-da-branch.
  3. Faça suas alterações e commits normalmente.
  4. Para unir a branch ao código principal, alterne para a branch principal com git checkout main ou git switch main.
  5. Realize o merge com git merge nome-da-branch.
  6. Se houver conflitos, resolva-os manualmente e finalize o merge.
